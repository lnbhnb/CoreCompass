# 校园二手交易平台 · 需求分析

## 1. 项目背景

校园内学生之间存在大量二手物品交易需求（教材、电子产品、生活用品等），但目前主要依赖微信群和贴吧，信息分散、交易效率低、缺乏信任机制。

## 2. 目标用户

- **买家：** 寻找低价优质二手物品的学生
- **卖家：** 希望快速出手闲置物品的学生
- **管理员：** 平台运营方，负责审核和纠纷处理

## 3. 核心功能

### 3.1 用户模块
- 注册登录（学号认证）
- 个人中心（信用评分、交易历史）

### 3.2 商品模块
- 发布商品（图片、描述、价格）
- 分类浏览和搜索
- 收藏和关注

### 3.3 交易模块
- 下单与支付（模拟）
- 订单状态跟踪
- 评价系统

### 3.4 安全模块
- 实名认证
- 举报与封禁
- 交易纠纷处理

## 4. 非功能需求

- 响应时间：核心接口 < 200ms
- 并发支持：100 QPS
- 数据安全：用户信息加密存储

## 5. 里程碑规划

- M1：数据库设计（1 周）
- M2：核心 API 实现（2 周）
- M3：前端页面（1.5 周）
- M4：测试与部署（1 周）

## 6. 风险评估

- 学号认证接口可能需要学校授权
- 支付模块涉及合规，Demo 阶段用模拟
