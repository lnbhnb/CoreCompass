# CoreCompass 录屏操作指引

> 录屏前请先完成【准备工作】。录屏时按【录制流程】操作，每一步有对应解说词。

---

## 准备工作（录屏前 5 分钟）

### 1. 配置 DeepSeek Key
打开 `backend/.env`，填入你的 DeepSeek key：
```
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx
```
（其他配置已就绪：飞书 webhook、DeepSeek base_url 等）

### 2. 启动后端
```powershell
cd d:\Desktop\CoreCompass\backend
uvicorn app.main:app --port 8000
```
看到 `Application startup complete` 即可。

### 3. 打开浏览器
访问 http://localhost:8000
应看到创建项目表单。

### 4. 打开飞书群
打开你配置了 webhook 的飞书群，待命切屏。

### 5. 准备测试文件
测试文件已备好，路径：`参赛材料/demo-files/`
- `empty.sql` —— 内容只有注释，用于演示失败
- `valid.sql` —— 含 3 表 + 主键 + 外键，用于演示通过
- `valid.md` —— 含 500+ 字 + H2 标题，用于演示 MD 校验

### 6. 录屏工具设置
- 分辨率：1280x720
- 工具：OBS / Windows 游戏栏（Win+G）
- 时长目标：5 分钟以内

## 录制流程（5 分钟脚本）

### 0:00-0:30 开场
**画面：** 浏览器停在 http://localhost:8000 创建项目页
**解说词：**
> 大家好，这是 CoreCompass，校园项目"伪需求"粉碎机。
> 学生拿到大课题往往无从下手，而 AI 聊天工具只会附和，不把关真实进度。
> CoreCompass 通过三大突出点解决这个问题。

### 0:30-1:30 段 1 · 项目拆解
**画面操作：**
1. 在表单填入：
   - 项目名：`校园二手交易平台`
   - 截止日期：`2026-08-20`（或任意未来日期）
   - 团队人数：`3`
   - 课题描述：`校园二手交易平台，支持学生发布闲置物品、浏览搜索、模拟交易`
2. 点【生成拆解计划】按钮
3. 等待 5-15 秒（LLM 调用）
4. 页面跳转到任务看板，展示 4-6 个里程碑卡片

**解说词：**
> 输入课题和团队信息，Agent 调用大模型生成里程碑和周任务计划。
> 看，自动拆出了数据库设计、核心实现、测试部署等里程碑，每个里程碑下有 core 和 optional 任务。

### 1:30-2:30 段 2 · 突出点① 硬验收
**画面操作：**
1. 找到第一个里程碑卡片（"数据库设计"或类似名称，expected_artifact_type=sql）
2. 点上传按钮，选择 `参赛材料/demo-files/empty.sql`
3. 看到红色"✗ 未通过"提示，列出失败原因（缺少外键等）
4. 里程碑状态变 `locked`
5. 再次点上传，选择 `参赛材料/demo-files/valid.sql`
6. 看到绿色"✓ 通过，解锁下一阶段"
7. 里程碑状态变 `done`

**解说词：**
> 这是突出点①：硬验收。Agent 不盲信用户，强制校验真实产物。
> 上传空 SQL 文件，Agent 检测到缺少外键，红色拒绝，里程碑进度被锁定。
> 换成合规的 SQL（含表、主键、外键），绿色通过，自动解锁下一阶段。
> 这在 Notion、Jira 里都没有，它们只存文件不校验。

### 2:30-3:30 段 3 · 突出点② 动态重算
**画面操作：**
1. 点页面顶部的【模拟偏航】按钮
2. 看板上有任务状态变成 `overdue`（红色）
3. 点【触发重规划】按钮
4. 弹出模态框，显示：
   - 缺口 X 人天
   - LLM 提案：建议砍掉哪些任务（任务 ID）
   - LLM 提案：建议降级哪些任务
5. 点【确认应用】
6. 看板刷新，被砍的任务消失或标记为 `cut`

**解说词：**
> 这是突出点②：动态重算，就像 GPS 检测堵车会重规划路线。
> 点"模拟偏航"，Agent 把进行中任务标为逾期。
> 点"触发重规划"，Agent 计算产能缺口，调用 LLM 生成砍需求提案。
> 关键铁律：只能砍 optional 任务，core 任务只能降级，不能砍。这保证了项目核心目标不丢失。

### 3:30-4:30 段 4 · 突出点③ 主动打扰
**画面操作：**
1. 滚动到页面底部"通知日志"面板
2. 点【测试推送飞书】按钮
3. 切屏到飞书群
4. 飞书群几秒内收到 Agent 消息："【CoreCompass 测试通知】项目 ID=X 的 Agent 正在主动联系你..."
5. 切回浏览器，通知日志面板出现一条 `sent` 记录

**解说词：**
> 这是突出点③：主动打扰。
> 飞书、钉钉的提醒都靠人触发，CoreCompass 的 Agent 是主动的。
> 点"测试推送飞书"，几秒内群内收到 Agent 催办消息。
> 实际部署时，APScheduler 会每 60 分钟自动扫描逾期任务并推送，无需人工触发。

### 4:30-5:00 结尾
**画面：** 看板全景
**解说词：**
> 总结：CoreCompass 通过硬验收、动态重算、主动打扰三大突出点，
> 让学生团队的项目管理从"AI 附和"升级到"Agent 把关"。
> 技术栈是 FastAPI + SQLite + Alpine.js + DeepSeek + 飞书，
> 29 个测试全绿，可推广到所有校园项目团队。谢谢。

## 录屏后处理

### 压缩到 50MB 以下
```powershell
ffmpeg -i raw.mp4 -vcodec libx264 -crf 28 -preset slow -acodec aac -b:a 96k "参赛材料/CoreCompass-录屏.mp4"
```
如果没装 ffmpeg，可以用 HandBrake（图形界面）：https://handbrake.fr/

### 验证文件大小
```powershell
Get-Item "参赛材料\CoreCompass-录屏.mp4" | Select-Object Name, @{N='MB';E={[math]::Round($_.Length/1MB,1)}}
```
应 ≤ 50MB。

## 常见问题

**Q: LLM 调用失败怎么办？**
A: 检查 DeepSeek key 是否正确、网络是否通。失败时会自动用 fallback 4 里程碑模板，Demo 仍可跑。

**Q: 飞书推送失败？**
A: 检查 webhook URL 和 secret 是否匹配。失败时通知日志会显示 `failed`，但 Demo 流程不受影响。

**Q: 上传 SQL 一直不通过？**
A: 确认用的是 `参赛材料/demo-files/valid.sql`，它包含表、主键、外键，符合校验规则。

**Q: 录屏卡顿？**
A: 关掉其他重型程序，OBS 编码器选 x264，目标 720p 30fps。
